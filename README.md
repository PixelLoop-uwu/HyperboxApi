# ☃️ HyperBox API

**Backend API для проекта HyperBox**

Реализован на `FastAPI` и предоставляет методы авторизации, управления скинами, получения модпаков и вспомогательные ресурсы.

## Технологии

- Python 3.10+
- FastAPI
- SQLAlchemy (async) + aiosqlite
- bcrypt, loguru, aiohttp

## Архитектура

### Структура репозитория
- `api/` — исходники сервера
  - `main.py` — точка входа приложения
  - `routes/` — API маршруты (auth, modpacks, resources, skins)
  - `services/` — бизнес-логика (auth, skins)
  - `database.py` — менеджер БД с SQLAlchemy
  - `models.py` — Pydantic модели для валидации
  - `utils.py` — вспомогательные функции (хеширование, генерация токенов)
  - `bases.py` — ORM модели
- `DATA/` — статические данные: modpacks, libraries, Java JDK

### Основные компоненты

**Авторизация:**
- Регистрация и вход пользователей
- Генерация и хранение токенов
- Проверка учётных данных с bcrypt
- Выдача assets_token для клиентов

**API эндпоинты:**
- `/users/login` — аутентификация пользователя
- `/users/register` — регистрация нового пользователя
- `/users/delete` — удаление пользователя (admin)
- `/skins/*` — управление скинами
- `/modpacks/*` — получение данных модпаков
- `/resources/*` — вспомогательные ресурсы

**База данных:**
- SQLite с асинхронным драйвером aiosqlite
- ORM через SQLAlchemy
- Автоматическое создание таблиц

## Ключевые файлы

- `api/main.py` — инициализация FastAPI, регистрация маршрутов
- `api/services/auth.py` — логика авторизации и управления токенами
- `api/database.py` — работа с БД и кеширование
- `api/models.py` — Pydantic схемы для запросов и ответов
- `api/utils.py` — хеширование паролей, генерация токенов

## Особенности

- Асинхронная архитектура с async/await
- Логирование через loguru
- Безопасное хранение паролей через bcrypt
- Поддержка переменных окружения (.env)
- Интеграция с фронтенд-клиентами и лаунчером
